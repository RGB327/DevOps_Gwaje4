# AWS

## 클라우드의 의미

> 인터넷에 연결된 가상화된 서버를 통해 서버, 스토리지, 데이터베이스 등의 컴퓨팅 자원을 빌려 쓰는 서비스

**특징 및 장점**
- 비용 효율성
- 유연성 및 확장성
- 접근성
- 유지 보수

**클라우드 서비스 모델**

- **IaaS(Infrastructure as a Service)**
    - 서버나 저장소 등 인프라를 제공
    - 예) AWS EC2
- **PaaS(Platform as a Service)**
    - 애플리케이션 개발/실행 환경 플랫폼 제공
    - 예) AWS Elastic Beanstalk
- **SaaS(Software as a Service)**
    - 설치 없이 인터넷으로 소프트웨어를 바로 사용
    - 예) Google Drive, Netflix

**배포 형태에 따른 구분**
- 퍼블릭 클라우드
    - AWS, Google cloud 등 서비스 제공업체가 소유한 인프라를 여러 사용자에게 공유하는 모델
- 프라이빗 클라우드
    - 특정 기업이나 사용자에게만 공유되는 모델
    - 보안이 뛰어남
- 하이브리드 클라우드
    - 민감한 데이터는 프라이빗에 일반 업무는 퍼블릭으로 분리해 사용하는 방식
    - 비용 절감에 용이함

## AWS 인프라 환경

#### 리전 (Region)
> AWS의 서비스가 운영되는 지리적인 영역, 세계 곳곳에 흩어져있는 거대한 물리적 위치를 뜻함

**리전의 특성**

- **독립성**
    - 각 리전은 서로 독립되어있어 한 리전에서 문제가 생겨도 다른 리전에는 영향을 주지 않음

- **리소스 위치**
    - 사용자는 지연 시간을 줄이거나 법적 규제를 준수하기 위해 특정 리전을 선택해 리소스를 생성함

하나의 리전은 반드시 3개 이상의 가용 영역으로 구성됨

#### 가용 영역(Availability Zone)
> 하나의 리전 내에서 물리적으로 분리된 하나 이상의 데이터 센터들의 집합

**가용 영역의 특성**
- **물리적 독립**
    - 각 가용 영역은 개별적인 전력, 냉각 네트워크 시설을 갖추고 있어 침수나 화재 같은 재해로부터 서로 보호됨
- **고가용성 구성**
    - 동일 리전 내 가용 영역들은 전용 고속 네트워크로 연결되어 있어 애플리케이션을 여러 AZ에 분산 배치하면 특정 데이터 센터에 문제가 생겨도 서비스 중단 없이 운영이 가능함

#### 엣지 로케이션(Edge Location)
> 전 세계 주요 도시에 설치된 캐싱 목적의 소규모 데이터 센터

**사용하는 이유**
사용자와의 물리적 거리를 줄여 지연 시간을 최소화하기 위해

**주요 서비스**
- **CDN(CloudFront)**
    - 정적 콘텐츠를 사용자 근처에 저장하여 빠르게 전달
- **DNS(Route 53)**
    - 전 세계에서 빠른 DNS 지원
- **AWS WAF/Shield**
    - 보안 공격을 네트워크 끝단에서 먼저 차단

## AWS Storage
> 데이터를 안전하고 효율적으로 관리하기 위해 사용하는 서비

예시: EBS , S3, EFS

**스토리지의 구성 요소**

- **DAS(Direct-Attached Storage)**
    - 가장 전통적인 방식의 스토리지로 서버에 직접 연결되는 외장형 저장 장치이다
    - 별도의 네트워크를 거치지 않아 설치가 간편하고 성능도 우수함

- **NAS(Network-Attached Storage)**
    - 네트워크를 통해 여러 서버나 사용자에게 파일을 제공하는 방식
    - 이더넷 기반으로 연결되며 일반적인 파일 공유와 백업 용도로 사용


- **SAN(Storage Area Network)**
    - 전용선인 고속 네트워크를 사용하는 확장 가능한 고성능 스토리지로  솔루션이 필요한 대기업에 적합
    - 데이터 복제, 스냅샷, 재해 복구 등의 고급 기능을 갖춘 중앙 집중식 관리
(비용이 많이 들고 관리가 복잡함)

## AWS Network

#### VPC (Virtual Private Cloud)
- 클라우드 내 사용자만 사용할 수 있도록 논리적으로 독립된 네트워크 공간

##### VPC의 핵심 구성 요소
- **IP주소 범위(CIDR)**
    - 네트워크의 주소 범위를 결정함
    - 앞부분은 네트워크의 시작 주소 뒷부분은 / 서브넷 마스크로 표시함
    - AWS의 예약 주소
    ```
    0: 네트워크 주소
    1: VPC 라우터
    2: DNS 서버
    3: 미래 예약용
    255: 미래 예약용
    ```

- **서브넷**
    - **public subnet**: 외부 사용자가 접속해야하는 웹서버 등을 둘 때 사용
    - **private subnet**: 외부에서 접속할 수 없는 서브넷으로 데이터베이스, 로그 등의 보안이 필요한 환경에서 사용

- **IGW(Internet GateWay)**
    - VPC 내부 리소스와 인터넷 간의 통신을 가능하게 해주는 인터넷 통로
    - **작동 방식:**
        1. Public IP와 Private IP를 매핑해놓음
        2. 인스턴스가 패킷을 보낼 때 Private IP를 보내고 IGW를 통과할 떄 Public IP로 바꾸어 전달
        3. 외부에서 퍼블릭 IP로 패킷을 보낼 시 연결된 프라이빗 IP로 바꿔 내부 전달
    - 수평적으로 확장되는 논리적 구성 요소로 병목현상이 거의 없음

- **라우팅 테이블**
    - 네트워크 장치(라우터, 컴퓨터)가 데이터 패킷을 목적지까지 가장 효율적인 경로로 전송하기 위해 참조하는 데이터베이스 리스트
    - AWS에서 외부와 내부 서브넷을 나눌 떄 사용

## AWS 아키텍쳐
> 가용성, 확장성, 보안을 극대화하도록 설계된 클라우드 기반의 인프라 구조

방식:
**3-Tier Architecture**
- 가장 표준적이고 안정적인 방식
- 웹 서버, 애플리케이션 서버, 데이터베이스를 물리적/논리적으로 분리
- **구성**: ALB (로드 밸런서) + EC2 (가상 서버) + RDS (데이터베이스)

**Serverless Architecture**
- 서버를 직접 관리하지 않고 이벤트가 발생할 때만 코드가 실행되게 하는 방식
- **특징**: 사용한 만큼만 비용을 내고, 관리 부담이 거의 없지만 실행 시간이 긴 작업에는 부적합
- **구성**: API Gateway + Lambda + DynamoDB

## AWS 서비스


**Computing Service**
- **EC2
(Elastic Compute Cloud)**
    - 가상 서버를 생성해 리눅스나 윈도우 환경을 제공

- **AWS Lambda**
    - 서버 관리 없이 코드를 실행하는 서버리스 컴퓨팅 서비스

**Networking Service**
- **Amazon VPC(Virtual Private Cloud)**
    - AWS 클라우드 내에 논리적으로 격리된 가상 네트워크를 구축
- **Amazon CloudFront**
    - 빠르고 안전하게 콘텐츠를 배포하는 CDN(Content Delivery Network) 서비스
- **Route 53**
    - DNS 서비스로 도메인 구입부터 네임서버 등록까지 DNS에 필요한 모든 기능이 있고 추가로 모니터링 기능까지 지원하는 서비스
- **ELB(Elastic Load Balancing)**
    - 많은 접속자를 여러 대의 서버로 공평하게 나눠주는 서비스
    - VPC에 탑재되며 트래픽을 VPC 내 리소스들(EC2 등)에 부하 분산시킴
